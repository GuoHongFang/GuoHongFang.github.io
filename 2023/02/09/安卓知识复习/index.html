关于安卓基础知识复习

发表于2023-02-09|更新于2023-02-15|Android
|阅读量:

走进安卓

  • 移动电商，全称移动电子商务，是以无线终端为载体的电商模式

  • 移动电商具有方便，安全，易于推广，开放性，包容性等特点

  • 移动电商可以提供购物订票交易娱乐等服务。为生活提供了

  • 移动电商主要依靠移动电子设备和移动互联网逐渐兴起，并在生活中被人们广泛运用。

  • Linux内核层：安卓操作系统是基于linux内核。

  • android系统的四大优势：

    1. 开源性
    2. 丰富的硬件选择
    3. 不受任何限制的开发商
    4. 无缝衔接的goole应用

  • android 是一种以Linux和Java为基础的开源代码操作系统，主要适用于便携设备。

  • android系统最初由Andy Rubin开发，2005年8月17被goole收购。

  • android系统架构的四层机构，从上层到下层分别是应用程序层，应用程序框架层，系统运行库层，linux内核层

  • 真机调试的方法：

    首先，使用use线连接计算机与手机，有些手机或自动获取并安装驱动，不行的话可以借助第三方助手下载安装驱动，其次，打开手机的use调试功能，打开手机的开发者选项，勾选“use调试”。

  • android studio是由goole公司基于intellij IDEA开发的

  • android默认采用gradle工具构建项目

  • android studio是跨平台的android开发环境，支持windows，mac，linux系统

  • jdk全称java development kit，jre全称 Java running environment。

  • 如果你的电脑是Intel处理器的话，那么Intel HAXM的加速技术，会让模拟器在启动和执行速度上有很大的提升。

架构设计

  • 数据结构设计：

    1. 网络存储：通过网络获取远端服务器的数据

    2. sharedPreferences参数存储：以键值对形式存储在xml中，应用场景：轻量级存储

      存储数据保存数据一般分为四个步骤：

      • 使用Activity类的getSharedPreferences方法获得SharedPreferences对象；

      • 使用SharedPreferences接口的edit获得SharedPreferences.Editor对象；

      • 通过SharedPreferences.Editor接口的putXXX方法保存key-value对；

      • 通过过SharedPreferences.Editor接口的commit方法保存key-value对。

        读取数据读取数据一般分为两个步骤：

      • 使用Activity类的getSharedPreferences方法获得SharedPreferences对象；

      • 通过SharedPreferences对象的getXXX方法获取数据；

    3. SqlLite数据库存储：嵌入式数据库

    4. 文件存储:通过java io的读写方法存储

    5. ContentProvider存储：四大组件之一，作为传输数据的媒介

  • 资源文件夹

    1. anim：自定义的动画资源文件。
    2. drawable：各类选择器和选择器有关的图片。
    3. layout：布局文件。
    4. values/values-w820dp:样式尺寸文字颜色资源等。

  • 关于activity的生命周期：（7个 关键方法）

    1. onCreate（）：在activity第一次创建时调用
    2. onStrart（）：当Activity第一次变得可见时调用
    3. onRestart：在一个Activity再次启动时调用
    4. onResume：当Activity在用户进行交互时调用
    5. onPause：当一个系统即将调用另外一个Activity时，当前Activity会先调用onPause（）
    6. onStop（）：当Activity变得不可见时调用
    7. onDestroy：当前Activity被销毁

  • 思考，ActivityA跳转到ActivityB，然后返回到ActivityA，最后在推出ActivityA,ActivityA和ActivityB的生命周期怎么变化？

    依次执行以下方法：onpause（跳转），onstop（A不可见）,onRestart(A再次启动)，onstart（），onResume（）

  • 创建app完整页面的三个步骤：

    1. 在layout目录创建xml文件
    2. 创建与xml文件对应的Java代码
    3. 在AndroidMainfest.xml中注册配置

网络层和数据模型的封装

  • 什么是RestFul Api？

    1. Rest，即Representational state transfer的缩写，为表现层状态转换。Restful api 是一种新的风格的api，它定义为get/post/patch/put/delete分别对应数据库的操作
    2. RxJava是一个在Java VM上使用可观测的序列来组成异步的、基于事件的程序的库。

  • 使用okhttp和thhp Get 请求的步骤

    1. 第一步，新建一个okhttpClient对象，这个对象最好在整个app中只有一个实例存在
    2. 第二步，新建一个request请求对象，它使用Builder的方式进行创建，需要包含请求的接口url
    3. 第三步，使用okhttpclient对象创建一个call请求，并传入request对象
    4. 第四步，将call加入请求队列，执行该请求。

  • Gson框架解析

    1. 将java对象转化为JSON字符串

    2. 生成单个实例类对象。

    3. 生成List集合

  • 理解MVP架构模式

    mvp模式，它是由mvp模式演变而来的，他们的基本思想有相通的地方：Model负责提供数据，View负责提供显示，Controller/Persenter负责业务逻辑的处理。

    • (区别)在mvp模式中Persenter将View和Model完全分离，View是不可以直接使用Model的，他们之间的通信通过presenter来进行，所有的交互逻辑都是在Presenter的内部，而在mvc模式中，View可以直接从Model中读取数据

  • 使用GreenDao进行数据库的增删改查

    1. 进行操作之前需获取一个GreenDao的实例

  • Retrofit默认使用httpok作为网络层

    get请求：使用Retrofit框架进行网络请求时，需要提供网络请求业务方法，包括URL，请求方式，参数和返回值类型

  • Android四大组件

    1. Activity 页面
    2. Service 服务
    3. Content Provider 内容提供者
    4. BroadCast receiver 广播者

  • Rxjava转换概念中map和flatmaap的比较

  1. 相同点：把传入的参数转化之后返回另一个对象
  2. 不同点：map是把发射对象转化为另一个对象发射出去，flatmap是把发射对象转化成另一个observable对象，进而把这个observable发射的对象发射出去。
  • 命名空间 说明
    com.huatec.edu.mobileshop.activity 与用户界面相关的源代码文件
    com.huatec.edu.mobileshop.adapter 与数据适配器相关的源代码文件

    com.huatec.edu.mobileshop.common 通用类源代码文件，如常量类、Application子类

    com.huatec.edu.mobileshop.server 与Server相关的服务类源代码文件
    com.huatec.edu.mobileshop.util 封装的一些工具类源代码文件
