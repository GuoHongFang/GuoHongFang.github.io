继承：

  • ​ 把每个类中相同的成员属性和方法抽取到一个单独的类中，该类就是父类，而其它类可以通过extends关键字继承

多态

  • 概念：同一个对象表现出不同的形态

  • 例如：狗就是狗，也是动物，动物 Animal = new 狗；

  • 条件：①有实现或者继承的关系②方法的重写，子类重写父类③父类指向子类

  • 多态访问特点：①成员变量：编译看左边，执行看左边 ②成员方法：编译看左边，执行看右边

  • 优点：提高了程序的拓展性

  • 具体实现：定义方法的时候，使用父类作为参数，在将来使用的时候，使用具体的子类参与操作

  • 弊端：不能使用子类特有的功能

抽象类的特点：

  • 抽象类中不一定有抽象方法，但是有抽象方法的类一定的抽象类
  • 抽象类不能实例化，必须通过子类实例化
  • 子类必须对父类的方法进行重写
  • 在抽象类中可以定义非抽象方法
  • 如果子类不想重写父类中的抽象方法，那么该子类必须为抽象类
    • 抽象类中可以有构造方法，用于子类访问父类时的数据初始化

多线程

什么是线程安全问题？

多线程同时对一个全局变量做写操作，可能会受到其他线程的干扰，就会发生线程安全问题

如何解决线程安全问题？

  • 核心思想：上锁 分布式锁
  • 在jvm中，多个线程需要去竞争锁的资源，最终只有一个线程能获取到锁，哪个线程获取到锁哪个线程就执行代码，如果没有获取成功，则中间需要经历锁的升级 过程
  • 如果一直没有获取到锁，则会一直阻塞等待
  • 但是如果A获取锁成功，却一直不释放锁，那先线程B则一直阻塞等待。

  • 如果synchronized锁加载在我们实例方法上，使用this锁

  • 如果synchronized加载在我们静态方法上，使用当前类名.class方法

  • public static void cal(){
    synchronized (ThreadCount.class){
        count--;
        System.out.println(Thread.currentThread().getName()+","+count);
    }

}
